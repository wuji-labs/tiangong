# 例 03 · 设计评审(Design Review · 用六法尺评一版"没灵魂"的 Dashboard)

锁定本 skill 在「评审一版技术上没错但没灵魂的设计」场景下,如何用六法给出可落地的诊断而非"感觉不太行"。

---

## INPUT(用户请求)

> 这版数据 dashboard 设计师说"完全按规范做的,网格、间距、组件都对",但老板看了说"没感觉、像后台"。到底差在哪?帮我评审,要能落地改。

## 没有本 skill 的产出(反模式)

```
嗯……整体挺干净的,配色可以再活泼一点?
要不加个渐变背景、换个更现代的字体、加点动效?
（"感觉"层面的含糊建议,无尺度、无优先级,改完大概率还是没灵魂)
```

## 有本 skill 的产出(用六法尺逐项诊断)

```
[天工 | 目标:评审"没灵魂"dashboard | 性:数据——清、可信、可读久 | 诊断:气韵死 + 留白欠 + 骨法平 | 优先级:先经营位置,再骨法,后气韵]
```

### 逐六法过尺(每条给:现状判定 → 病灶 → 落地改法)

| 法 | 现状判定 | 病灶 | 落地改法 |
|----|---------|------|---------|
| 气韵生动 | ✗ 死 | 满屏等权卡片,视线无处落,去掉文字后构图空洞 | 定一个"主景"(核心指标占视觉重心),其余退为配景——视线先有落点才有流动 |
| 骨法用笔 | ✗ 平 | 所有卡片同字重同大小同灰度,无轻重层级 | 用墨分五色:核心数字焦(#1A1A1A),次级重,辅助清——力度拉开 |
| 应物象形 | △ 半 | 用了通用后台模板,没问"数据 dashboard 之性" | 数据之性 = 久读不累。降饱和、留白、字体选可读性强者(骨力非花哨) |
| 随类赋彩 | ✗ 杂 | 红黄蓝绿各色图表并置,无主次 | 限色:一套同源色阶担所有图表,仅"警戒"用一点朱砂强调 |
| 经营位置 | ✗ 欠 | 每像素填满,卡片紧贴,无留白,密不透风 | 拉间距至基准 8px 的 3–4 倍,疏密有致:关键区密、外围疏;留白升至约 40% |
| 传移模写 | △ | 只临了 Material,未临东方留白与宋之素雅 | 参宋代美学(素雅、理趣):减装饰、加呼吸,让"理"自己说话 |

### 根因排序(先动哪一刀)

老板说的"没灵魂",根因不是"不够活泼",恰恰相反:
1. **首病在经营位置**(留白欠)——密不透风,所以窒息。先拉留白,呼吸一开,气韵自来。
2. **次病在骨法**(无层级)——等权 = 无重点。用墨分五色拉开力度。
3. **末病才是气韵**——前两者修好,主景一立,气韵是结果而非起点。

> ✗ 反向药方:加渐变 / 加动效 / 换花哨字体——是在"密不透风"上再堆料,越救越闷(大巧若拙的反面)。

### 一句话给老板

> 不是不够花,是**不够空**。留白拉开、层级拉开、主景立住——它就活了。

---

**要点**:评审"没灵魂"不靠感觉,靠六法尺逐项判定 + 根因排序。
"没灵魂"的真因常是**留白欠 + 层级平**,药方是减与空(经营位置 + 骨法),而非加与花。
